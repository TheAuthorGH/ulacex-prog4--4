Java - Características y funcionamiento

Características de filosofía de diseño:

  • Tipado fuerte: Todos los datos manejados en Java se manejan con tipado fuerte. Es decir, que los tipos se deben saber antes de la compilación y ejecución.
  • Clases, Herencia, Encapsulacion: Java intenta seguir varias decisiones de diseño tomadas por lenguajes orientados a objeto tradicionales e intenta llevarlas al máximo.
  • Write once, run anywhere: Java intenta ser completamente independiente de la plataforma en la que corre, lo cual logra por medio de varias implementaciones del Java Virtual Machine.
  • Compilador inteligente: Java es un leanguaje compilado, y el compilador permite encontrar varios errores antes de ejecución.

Ventajas de Java:

Nota importante: La lista a continuación es de carácter subjetivo. No hay tal cosa como una lista de ventajas o desventajas de una tecnología que sea considerada cien por ciento cierta.

  • Tipado fuerte contribuye al desarrollo: El tipado fuerte permite prevenir errores y crear código más explicito
  • Independiente de plataforma: Esto permite desarrollar fácilmente para varias plataformas, asegurándose de que no serán necesarios tantos cambios para cada plataforma.

Desventajas de Java:

  • Verbosidad: Java es uno de los lenguajes modernos más verbosos que hay. Es decir, hay que escribir más código para lograr lo mismo.
  • Orientación a objeto excesiva: Varios expertos opinan que los lenguajes como Java, que demandan el uso de clases y orientación a objeto, causan que los desarrolladores creen antipatrones, como por ejemplo, el abuso de la herencia, que en la actualidad no se considera una solución preferible en la mayoría de los casos.
  • Ambiente de desarrollo: Se require demasiada preparación de ambiente de desarrollo para cada proyecto de Java.

Java Virtual Machines

Cada implementación del JVM está diseñada para una plataforma distinta. Todo el bytecode, o código pre-compilado de Java, puede correr en cualquier JVM, idealmente (en la práctica, esto no es siempre cierto, pero esto se sale del alcance del tema). Actualmente existen implementaciones del JVM para todos los sitemas operativos importantes, incluyendo Linux, Mac OS, y las versiones más recientes de Windows.

La diferencia entre un archivo .java y .class es que un archivo .java es código no pre-compilado, legible por un humano. Un archivo .class contiene bytecode que está listo para ser ejecutado por un JVM, pero no puede ser modificado como código fuente.
